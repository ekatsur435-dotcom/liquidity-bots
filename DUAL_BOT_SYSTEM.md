### Скриншот 1-2: **DUMP Signals (SHORT BOT)**
- Это существующая система для поиска перекупленных монет
- Сигналы на SHORT (шорт)
- Паттерны: REJECTION SHORT, TRAP LONG, MEGA SHORT, DISTRIBUTION
- Работает на основе дельты, объёма, фандинга

- Показывает уровни (код 71/72, пик2/3)
- Это SMC (Smart Money Concepts) анализ
- Может использоваться для обоих ботов

### Скриншот 4: **Telegram канал DUMP Signals**
- Реальные сигналы в Telegram
- SHORT-зоны, плечо, цели ROI
- Интеграция TradingView → Webhook → Telegram/Бот

## 🎯 ДВЕ СИСТЕМЫ: LONG + SHORT

### 1. 🔴 SHORT BOT (DUMP Signals)
**Назначение:** Продавать перекупленные монеты (памп выдыхается)

**Сигналы:**
- RSI > 70 (перекупленность)
- Фандинг высокий положительный (лонги платят)
- L/S Ratio > 65% лонгов (толпа в лонгах)
- OI растёт с ценой (перегруз лонгов)
- Дельта отрицательная на росте (скрытые продажи)

**Паттерны:**
- 🔴 REJECTION SHORT — отбой от сопротивления
- 🔴 TRAP LONG — ловушка для лонгистов  
- 💥 MEGA SHORT — доминация продавцов
- 🟣 DISTRIBUTION — крупный игрок распродаёт

**Вход:** Шорт (продажа вниз)
**SL:** Выше уровня сопротивления
**TP:** Ниже (1.5%, 3%, 5%, 6.3%, 8.5%, 12.2%)

---

### 2. 🟢 LONG BOT (Liquidity Long Bot)
**Назначение:** Покупать перепроданные монеты (паника, дамп)

**Сигналы:**
- RSI < 30 (перепроданность)
- Фандинг отрицательный (шорты платят)
- L/S Ratio < 35% лонгов (толпа в шортах)
- OI растёт при падении (перегруз шортов)
- Дельта положительная на падении (скрытые покупки)

**Паттерны:**
- 🟢 REJECTION LONG — отбой от поддержки
- 🟢 TRAP SHORT — ловушка для шортистов
- 💥 MEGA LONG — доминация покупателей
- 🟣 ACCUMULATION — крупный игрок накапливает

**Вход:** Лонг (покупка вверх)
**SL:** Ниже уровня поддержки
**TP:** Выше (1.5%, 3%, 5%, 6.3%, 8.5%, 12.2%)

```
liquidity-dual-bot/
├── main.py                   # Точка входа, выбор режима
├── config.py                 # Настройки для обоих режимов
│
├── core/
│   ├── scanner.py            # Универсальный сканер

│   ├── short_mode.py         # Логика SHORT
│   ├── long_mode.py          # Логика LONG
│   └── dual_mode.py          # Оба направления одновременно
│
├── data/                       # API клиенты
├── risk/                       # Риск-менеджмент
├── execution/                  # Исполнение
└── bot/                        # Telegram
```

**Режимы работы:**
```bash
# Только SHORT
python main.py --mode short

# Только LONG
python main.py --mode long

# Оба направления (независимо)
python main.py --mode dual

# Геджирование (противоположные позиции)
python main.py --mode hedge
```

**Плюсы:**
- Один сервер, один код
- Общие ресурсы (Telegram, БД)
- Можно переключать режимы
- Риск-менеджмент общий (не превысить общий риск)

## 🎮 РЕЖИМЫ РАБОТЫ

### 1. Independent Mode (независимый)
```
SHORT Bot: BTC шорт (Score 72%)
LONG Bot:  ETH лонг (Score 68%)

Никакой связи между ними
```

### 2. Hedge Mode (хеджирование)
```
Если оба бота дают сигнал на одну монету:
- SHORT говорит: BTC перекуплен → шортить
- LONG говорит: BTC перепродан → лонгить

Решение: Не входить (конфликт сигналов)
Или: Меньший размер для обоих (стратегия хеджа)
```

### 3. Market Regime Mode (режим рынка)
```
Если рынок бычий ( bullish ):
  → Отключить SHORT бот
  → Только LONG бот работает

Если рынок медвежий ( bearish ):
  → Отключить LONG бот
  → Только SHORT бот работает

Если боковик ( sideways ):
  → Работают оба бота
```

## 💡 ОТВЕТЫ НА ВОПРОСЫ

### Q: TradingView нужен?
**A:** Нет, если используешь Python ботов. 
- Python боты получают данные напрямую с Binance/Bybit API
- Все индикаторы (RSI, OI, фандинг) считаются в коде
- TV можно использовать для визуализации, но не обязательно

### Q: А где SHORT бот из скриншотов?
**A:** На скриншотах — это существующая система (вероятно, Pine Script на TradingView + Webhook).
Мы можем:
1. Воссоздать её как Python бот
2. Интегрироваться с ней через webhook
3. Создать улучшенную версию с полными данными

### Q: Один сервер или два?
**A:** Один сервер, один бот с режимами:
```bash
python main.py --mode dual
```

### Q: Могут ли они работать одновременно?
**A:** Да! И это даже лучше:
- Диверсификация (не зависишь от одного направления)
- Хеджирование рисков
- Работа в боковике (когда оба направления дают сигналы)

### Q: Что если оба дадут сигнал на одну монету?
**A:** Возможны 3 стратегии:
1. **Priority:** Выбрать бота с более высоким Score
2. **Ignore:** Пропустить (конфликт сигналов)
3. **Hedge:** Входить в обе стороны с меньшим размером
